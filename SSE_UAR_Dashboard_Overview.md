## What Is This Dashboard?

The **SSE APM-to-AD Group Intelligence Pipeline** is a self-contained, browser-based tool that gives the SSE team a real-time view of every application (APM) under User Access Review (UAR). It answers one question for each app:

> *"Do we have MAR evidence to close this UAR finding — or do we need to take action?"*

The dashboard splits all active UAR APMs into actionable tabs: **Can Close** (evidence ready), **Issue / Needs Work** (follow-up required), **APM Universe** (all 7,952 APMs), and **Trends** (daily count history), with one-click CSV download and direct AuditBoard links for each finding.

---

## How the Report Is Built

The dashboard is generated by running a single Python script (`build_dashboard.py`) which orchestrates four data pipelines and produces one self-contained HTML file (~4 MB) with all data embedded.

### Step-by-Step Pipeline

```
1. MAR Files (Local CSV)
   └─ Loads ~752 MB of quarterly entitlement data from SharePoint
   └─ Builds entitlement → quarter mapping
   └─ Primary evidence source: CONSOLIDATEDEntitlementReport (5).csv (FY27_Q1)

2. BigQuery: unified_findings  (authoritative source — 364 active UAR findings / 293 APMs)
   └─ Filter: norm_status = 'Active' AND (issue_sub_type OR title LIKE '%UAR%')
   └─ Provides: title, due date, due status, owner, email, task_reviewer

3. BigQuery: uar_findings_enriched  (LEFT JOIN on issue_id — 260/364 matched)
   └─ Adds: AD groups, LLM-generated intelligence fields

4. BigQuery: sse_apm_data_prod
   └─ APM sensitivity: Highly Sensitive / Sensitive / Non-Sensitive
   └─ PCI and SOX flags per application

5. BigQuery: uar_apm_enriched + uar_galaxy_enriched  (APM Universe — 7,952 APMs)
   └─ All APMs with AD groups, Azure pending signal from Galaxy subscriptions

6. Inline Strict AD→MAR Matching (token-indexed, fast)
   └─ Cross-references every AD group from BQ against MAR entitlements
   └─ 4-rule matching algorithm (boundary, endswith, core-name, prefix-stripped)
   └─ Result: which APMs have verifiable MAR evidence → drives Can Close tab

7. Daily History Snapshot (dashboard_history.json)
   └─ Appended on every rebuild (same-day overwrites)
   └─ Tracks: Total, Can Close, Partial, Needs Work, Past Due
   └─ Embedded in HTML → drives 📈 Trends tab
```

### Output

A single `sse_uar_dashboard.html` file — no server, no database, no login required. Open in any browser.

## Backend Connections

| Source | Type | What It Provides |
|--------|------|------------------|
| `sse_data_lake.unified_findings` | BigQuery | **Authoritative** — 364 active UAR findings, 293 APMs; title, due date, owner, email, task_reviewer |
| `sse_findings_enriched_data.uar_findings_enriched` | BigQuery (LEFT JOIN) | AD groups + LLM-generated fields (260/364 matched) |
| `snow_apm_data.sse_apm_data_prod` | BigQuery | APM data classification (sensitivity), PCI/SOX flags |
| `sse_findings_enriched_data.uar_apm_enriched` | BigQuery | Full 7,952 APM universe with AD groups |
| `sse_findings_enriched_data.uar_galaxy_enriched` | BigQuery | Azure subscription signal (azure_pending flag) |
| MAR CSV files | Local SharePoint sync | Quarterly entitlement evidence (~752 MB, 6 files) |
| `dashboard_history.json` | Local repo file | Daily snapshot history for Trends tab |

**BigQuery Project:** `infosec-compliance-auditboard`  
**Auth:** Google Application Default Credentials (ADC) — no hardcoded keys

## AI-Enriched Fields (LLM, Inline in BQ)

Every finding in `uar_findings_enriched` includes LLM-generated intelligence:

| Field | Description |
|-------|-------------|
| **SLA Status** | Compliant or Non-Compliant based on deadline analysis |
| **Sentiment** | Positive / Neutral / Negative — tone of comment history |
| **Summarized Comments** | Plain-English summary of all finding comments |
| **Last Reach Out** | Who sent the last message — Customer (app team) or SSE |
| **Ownership Change Requested** | Flags apps where owner reassignment was requested |
| **Customer Question** | Flags open questions from the app team |
| **SSE Reviewer** | SSE team member assigned to the finding |

## Key Benefits

### For the SSE Team
- **Instant evidence readiness** — know at a glance which APMs can be closed today vs. which need work
- **One-click MAR CSV download** — pre-filtered evidence file per APM, ready to upload to AuditBoard
- **Direct AuditBoard links** — orange button opens the exact finding record
- **Email integration** — purple button pre-populates an email to the remediation owner
- **Progress tracking** — check ✅ Done per APM; state persists in browser LocalStorage across sessions

### For Leadership
- **Live data** — re-run the script any time to refresh from BigQuery; no manual exports
- **Full filter set** — slice by VBU, SLA, sensitivity, due status, quarter, SSE reviewer, or free-text search
- **Past Due tile** — one click shows all overdue findings across both tabs
- **Exportable** — "Filtered CSV" or "All CSV" buttons export the current view for reporting

### Operational
- **Self-contained** — the HTML file has zero external dependencies; works offline once generated
- **No infrastructure** — no web server, no database, no login; share via email or Teams
- **Audit-ready** — embedded MAR evidence CSVs include user IDs, display names, certification dates, and statuses

## GitHub Workflow — How to Make Updates

### Branch Roles

| Branch | Rule |
|--------|------|
| `master` | 🌐 **Live** — only merge here when ready to publish to the dashboard link |
| `dev` | 🛠 **Your working branch** — all edits go here first |
| `baseline` | 🔒 **Never touch** — frozen FY27 Q1 snapshot for audit reference |

### Day-to-Day Update Flow

```
dev  →  test locally  →  merge to master  →  push  →  live on GitHub Pages
```

**Step by step:**

```powershell
# 1. Switch to dev branch
git checkout dev

# 2. Make your changes (template.html, build_dashboard.py, etc.)

# 3. Rebuild and preview locally
py -3.13 build_dashboard.py
# Open sse_uar_dashboard.html in browser to verify

# 4. Commit to dev
git add build_dashboard.py template.html sse_uar_dashboard.html index.html
git commit -m "feat: describe what you changed"

# 5. Deploy to master (makes it live)
git checkout master
git merge dev
.\deploy.ps1        # builds fresh + pushes to both GHE and github.com
```

> **Shortcut:** `.\deploy.ps1` handles steps 3–5 automatically (build + commit + push).

### When to Use `baseline`
Only for **comparison** — never commit to it:
```powershell
git diff baseline master -- sse_uar_dashboard.html
```
Shows exactly what changed since the FY27 Q1 audit baseline.
